# PLAN

## Classification

- Size: small
- Domain Profile: general

## Phase 1

- Objective: 현재 프로젝트의 실제 범위와 운영 준비도를 파악한다.
- Tasks: 파일 인벤토리, 지침/메타데이터/자동화 설정 유무, Git 상태를 확인한다.
- Expected Files: 기존 콘텐츠 파일, .goal-harness 문서.
- Completion Criteria: 프로젝트에 존재하는 자산과 부재한 운영 구성요소를 구분했다.
- Test Point: 파일 목록 및 Git/실행 설정 정적 확인.
- Rollback/Recovery: 읽기 전용 검토이므로 되돌릴 프로젝트 변경이 없다.

## Phase 2

- Objective: 대표 콘텐츠의 품질·중복·운영 리스크를 분석한다.
- Tasks: 페르소나 항목 수, 필수 필드, 차별성, 게시 규칙·추적성 누락을 점검한다.
- Expected Files: 수다글_페르소나_20명.md.
- Completion Criteria: 근거가 있는 발견 사항을 심각도별로 정리했다.
- Test Point: 정적 텍스트 검사와 수동 표본 읽기.
- Rollback/Recovery: 콘텐츠를 수정하지 않고 발견 사항만 기록한다.

## Phase 3

- Objective: 검증 범위와 개선 우선순위를 제시한다.
- Tasks: 실행 가능한 검사 명령을 확인하고, 부재한 검증 체계는 한계로 명시한다.
- Expected Files: TESTS.md, 최종 검토 보고.
- Completion Criteria: 현 검토의 증거·한계·권장 다음 단계가 명확하다.
- Test Point: 가능한 정적 검사 실행 또는 N/A 사유 기록.
- Rollback/Recovery: 후속 구현은 별도 승인 후 진행한다.
